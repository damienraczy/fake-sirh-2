tables-gemini

Entreprise :
La table racine qui contient la description de l'entreprise simulée. C'est le point de départ de toutes les données.

Site_Géographique :
Liste les différents bureaux ou localisations de l'entreprise.

Unite_organisationnelle :
Définit les différentes unités organisationnelles (Ventes, IT, Marketing, RH, etc.). Chaque département est rattaché à un site.

Poste :
Catalogue de tous les rôles et intitulés de poste possibles dans l'entreprise (ex: "Développeur Senior", "Analyste Financier"). Cette table décrit la fonction, pas la personne qui l'occupe.

Employe :
La table centrale contenant les informations sur chaque personne (nom, prénom, identifiants). Elle contiendra une référence à un manager, qui est lui-même un autre employé.

Contrat :
Table essentielle qui lie un Employé à un Poste à un moment donné. Elle stocke le salaire, le type de contrat, les dates de début et de fin. Un employé peut avoir plusieurs contrats successifs au cours de sa carrière (promotions, changements de poste).

Évaluation_Performance :
Archive les résultats des entretiens de performance. Elle relie un employé, son évaluateur, une date, un score et des commentaires qualitatifs.

Compétence :
Une table de référence listant toutes les compétences identifiées dans l'entreprise (ex: "Python", "Négociation commerciale", "Management").

Employe_Competence :
Une table de jonction pour gérer la relation "plusieurs-à-plusieurs" entre les employés et les compétences. Elle indique quel employé possède quelle compétence, et éventuellement à quel niveau.

Historique_Carriere :
Un journal des événements importants pour chaque employé (embauche, promotion, changement de département, augmentation, départ). C'est crucial pour analyser les parcours et le turnover.
