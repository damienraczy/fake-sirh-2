Description du projet « Fake-sirh-2 »

Le projet Fake-sirh-2 consiste à concevoir un système d’information de gestion des ressources humaines factice. Contrairement à un véritable SIRH, ce système n’aura aucune interface utilisateur. Il reposera uniquement sur une base de données structurée, capable de simuler le fonctionnement organisationnel d’une entreprise.

Génération des données
	•	Les données de l’entreprise simulée seront générées à partir d’une description macroscopique de l’organisation (secteur, taille, structure, règles RH).
	•	Cette génération se fera via des commandes en ligne de commande (CLI) en Python.
	•	Les CLI s’appuieront sur Ollama Turbo (un moteur LLM récemment mis sur le marché, ollama.com) pour produire automatiquement les jeux de données.

Objectif principal

L’objectif de Fake-sirh-2 n’est pas de reproduire un SIRH opérationnel, mais de créer un corpus réaliste de données organisationnelles. Ces données serviront ensuite à :
	•	alimenter un système RAG (Retrieval-Augmented Generation),
	•	permettre aux entreprises de poser des questions et de simuler des scénarios RH ou organisationnels,
	•	tester des approches innovantes d’aide à la décision RH et de résolution de problématiques managériales.

Positionnement du projet
	•	Type : prototype expérimental de simulation RH.
	•	Usage : production de données synthétiques réalistes pour l’entraînement et l’évaluation de systèmes d’IA appliqués au domaine RH et organisationnel.
	•	Particularité : absence de front-end, interaction uniquement via la base et les CLI.
